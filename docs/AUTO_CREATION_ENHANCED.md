## Step 3: Prompt YAML 模板文件

### 目标
创建 10 个 YAML 模板文件，`BuiltinTemplateLoader` 启动时自动加载。

### 文件清单

所有文件位于 `src/main/resources/prompts/` 下：

#### 1. `WORLD_BUILDING/WRITING_RULES.yaml`
- 变量: `title, genre, description, worldSetting`
- 参数: maxTokens=2048, temperature=0.5
- 目的: 生成 10-15 条写作规则（叙事视角、文风约束、节奏规律、禁用套路）

#### 2. `WORLD_BUILDING/STYLE_FINGERPRINT.yaml`
- 变量: `title, genre, description, sampleContent`
- 参数: maxTokens=1024, temperature=0.5
- 目的: 提取风格指纹（句式特征、修辞偏好、情感表达方式）

#### 3. `CHARACTER_DESIGN/CHARACTER_BEHAVIOR_BOUNDARIES.yaml`
- 变量: `title, genre, cardContent, worldSetting`
- 参数: maxTokens=1024, temperature=0.5
- 目的: 定义角色行为边界（绝对不会、核心底线、应激反应）

#### 4. `OUTLINE_GENERATION/CHAPTER_EVENT_PLAN.yaml`
- 变量: `title, genre, chapterNumber, chapterOutline, previousEventPlans, characters`
- 参数: maxTokens=1024, temperature=0.6
- 目的: 设计章节事件序列（场景序列、冲突节点、情感弧线）

#### 5. `CHAPTER_WRITING/CHAPTER_CONTEXT_BRIEFING.yaml`
- 变量: `chapterNumber, chapterOutline, eventPlan, previousContent, characterStates, previousStoryline`
- 参数: maxTokens=1024, temperature=0.3
- 目的: 梳理前文关键信息（活跃悬念、情感基调、需延续细节）

#### 6. `CHAPTER_WRITING/CHAPTER_PLOT_REASONING.yaml`
- 变量: `chapterNumber, chapterOutline, contextBriefing, eventPlan, writingRules, behaviorBoundaries`
- 参数: maxTokens=1536, temperature=0.7
- 目的: 推演本章情节逻辑（核心冲突、转折验证、场景顺序）

#### 7. `CHAPTER_WRITING/CHAPTER_INSTANT_REVIEW.yaml`
- 变量: `chapterNumber, chapterContent, writingRules, styleFingerprint, behaviorBoundaries, plotReasoning`
- 参数: maxTokens=1536, temperature=0.2
- 目的: 即时质检（OOC检查、逻辑检查、风格检查、连续性检查）
- 输出格式: 【审查结论】通过/需优化 + 【问题清单】

#### 8. `CHAPTER_WRITING/CHAPTER_CONTENT_OPTIMIZATION.yaml`
- 变量: `chapterNumber, chapterContent, reviewFindings, plotReasoning`
- 参数: maxTokens=8192, temperature=0.5
- 目的: 根据审查意见修正正文（仅处理 Critical 问题）

#### 9. `CHAPTER_WRITING/CHAPTER_STORYLINE_UPDATE.yaml`
- 变量: `chapterNumber, chapterContent, previousStoryline, characterStates`
- 参数: maxTokens=1024, temperature=0.3
- 目的: 更新故事线快照（活跃伏笔、主线进展、待解决悬念）

#### 10. `CHAPTER_WRITING/CHAPTER_DEEP_REVIEW.yaml`
- 变量: `chapterRange, chapterSummaries, writingRules, styleFingerprint, activeStoryline`
- 参数: maxTokens=2048, temperature=0.4
- 目的: 跨章审查（角色一致性、伏笔进度、风格稳定性、节奏平衡）

## Step 5: EnhancedChapterWritingService

### 目标
封装单章 7 步精细创作循环逻辑。

### 新文件
`src/main/java/com/storycreator/workflow/autorun/strategy/EnhancedChapterWritingService.java`

### 核心方法

```java
@Service
public class EnhancedChapterWritingService {

    private final EnhancedSubStepExecutor executor;
    private final ChapterRepository chapterRepository;
    private final ChapterOutlineRepository chapterOutlineRepository;
    private final WorkflowEngine workflowEngine;
    private final WorkflowContextBuilder contextBuilder;

    /**
     * 单章 7 步精细创作循环
     */
    public void writeChapterEnhanced(AutoRunContext ctx, int chapterNumber,
                                      String writingRules, String styleFingerprint,
                                      int deepReviewInterval) {
        ChapterEntity chapter = loadChapter(ctx.getProjectId(), chapterNumber);

        // 断点续跑：根据 writingCycleStatus 决定从哪一步恢复
        String status = chapter.getWritingCycleStatus();

        // Step 1: 前文梳理
        if (isBeforeOrAt(status, "NOT_STARTED")) {
            String briefing = executor.generateContextBriefing(...);
            chapter.setWritingBriefing(briefing);
            chapter.setWritingCycleStatus("BRIEFING_DONE");
            save(chapter);
        }

        // Step 2: 剧情推演
        if (isBeforeOrAt(status, "BRIEFING_DONE")) {
            String reasoning = executor.generatePlotReasoning(...);
            chapter.setWritingReasoning(reasoning);
            chapter.setWritingCycleStatus("REASONING_DONE");
            save(chapter);
        }

        // Step 3: 正文生成（复用现有 chapter writing 逻辑）
        if (isBeforeOrAt(status, "REASONING_DONE")) {
            ctx.generateAndSave(WorkflowStep.CHAPTER_WRITING, chapterNumber, "精品模式");
            chapter.setWritingCycleStatus("CONTENT_DONE");
            save(chapter);
        }

        // Step 4: 即时审查
        if (isBeforeOrAt(status, "CONTENT_DONE")) {
            String review = executor.runInstantReview(...);
            chapter.setInstantReview(review);
            chapter.setWritingCycleStatus("REVIEWED");
            save(chapter);
        }

        // Step 5: 内容优化（条件执行）
        if (isBeforeOrAt(status, "REVIEWED")) {
            if (needsOptimization(chapter.getInstantReview())) {
                String optimized = executor.runContentOptimization(...);
                chapter.setContent(optimized);
                chapter.setWordCount(optimized.length());
            }
            chapter.setWritingCycleStatus("OPTIMIZED");
            save(chapter);
        }

        // Step 6: 故事线更新
        if (isBeforeOrAt(status, "OPTIMIZED")) {
            String storyline = executor.updateStoryline(...);
            chapter.setStorylineSnapshot(storyline);
            chapter.setWritingCycleStatus("STORYLINE_DONE");
            save(chapter);
        }

        // Step 7: 深度审查（周期性）
        if (isBeforeOrAt(status, "STORYLINE_DONE")) {
            if (chapterNumber % deepReviewInterval == 0 && chapterNumber > 0) {
                String deepReviewResult = executor.runDeepReview(...);
                chapter.setDeepReview(deepReviewResult);
            }
            chapter.setWritingCycleStatus("DONE");
            save(chapter);
        }

        // 生成角色状态（复用现有逻辑）
        workflowEngine.generateCharacterStates(ctx.getProjectId(), chapterNumber);
    }

    private boolean needsOptimization(String reviewResult) {
        return reviewResult != null && reviewResult.contains("需优化");
    }
}
```

## Step 6: EnhancedAutoRunStrategy 策略类

### 目标
实现 ENHANCED 策略的完整编排逻辑。

### 新文件
`src/main/java/com/storycreator/workflow/autorun/strategy/EnhancedAutoRunStrategy.java`

### 整体流程

```java
@Component
public class EnhancedAutoRunStrategy implements AutoRunStrategy {

    @Override
    public String getName() { return "ENHANCED"; }

    @Override
    public void execute(AutoRunContext ctx) throws Exception {
        Long projectId = ctx.getProjectId();

        // ═══════ Phase 0: PREPARATION ═══════
        runPreparation(ctx);

        // ═══════ 后续步骤：遍历 WorkflowStep ═══════
        WorkflowStep step = getStartStep(ctx);  // 从 currentStep 开始
        while (step != null) {
            if (ctx.shouldStop()) return;
            if (!ctx.isStepEnabled(step)) { step = step.next(); continue; }

            switch (step) {
                case WORLD_BUILDING -> runWorldBuilding(ctx);
                case CHARACTER_DESIGN -> runCharacterDesign(ctx);
                case OUTLINE_GENERATION -> runOutlineGeneration(ctx);
                case CHAPTER_WRITING -> runChapterWritingEnhanced(ctx);
                case POLISHING -> runPolishingEnhanced(ctx);
                case PROOFREADING -> runProofreading(ctx);
            }

            ctx.confirmStep(projectId, step);
            step = step.next();
        }
        // 标记完成
        markCompleted(ctx);
    }
}
```

### 各阶段详细逻辑

#### `runPreparation(ctx)`
```
1. 加载项目基本信息 (title, genre, description)
2. if writingRulesRepo.findByProjectId(id).isEmpty():
   - ctx.updateProgress(null, 0, "精品模式: 生成写作规则...")
   - 构建变量: title, genre, description, worldSetting(可能为空)
   - result = executor.generateWritingRules(projectId, vars, genre)
   - 保存到 writing_rules 表
3. if styleFingerprintRepo.findByProjectId(id).isEmpty():
   - ctx.updateProgress(null, 0, "精品模式: 提取风格指纹...")
   - 构建变量: title, genre, description, sampleContent(description 作为样本)
   - result = executor.generateStyleFingerprint(projectId, vars, genre)
   - 保存到 style_fingerprints 表
```

#### `runCharacterDesign(ctx)` — 增强部分
```
1. 复用默认逻辑：生成角色卡 + 精修
2. 新增：为每个 REFINED 且 behaviorBoundaries 为空的角色生成行为边界
   for each character:
     if shouldStop() return
     ctx.updateProgress(CHARACTER_DESIGN, 0, "精品模式: 生成行为边界 - " + name)
     result = executor.generateBehaviorBoundaries(projectId, vars, genre)
     character.setBehaviorBoundaries(result)
     save(character)
```

#### `runOutlineGeneration(ctx)` — 增强部分
```
1. 复用默认逻辑：4阶段大纲生成
2. 新增：为每个 eventPlan 为空的章节大纲生成事件计划
   for each outline:
     if shouldStop() return
     ctx.updateProgress(OUTLINE_GENERATION, chNum, "精品模式: 生成事件计划 第" + chNum + "章")
     result = executor.generateEventPlan(projectId, vars, genre)
     outline.setEventPlan(result)
     save(outline)
```

#### `runChapterWritingEnhanced(ctx)`
```
1. 加载 writingRules 和 styleFingerprint
2. 确定起始章节（同 DefaultAutoRunStrategy: 找第一个无内容或 writingCycleStatus != DONE 的章节）
3. for each chapter:
     if shouldStop() return
     ctx.updateProgress(CHAPTER_WRITING, chNum, "精品模式: 第" + chNum + "章写作循环")
     enhancedChapterWritingService.writeChapterEnhanced(ctx, chNum, rules, fingerprint, interval)
4. 生成标题（复用 workflowEngine.generateAndSaveTitles）
```

#### `runPolishingEnhanced(ctx)`
```
1. 加载 styleFingerprint
2. for each chapter (polishStatus != CONFIRMED):
     // 将风格指纹和审查 WARNING 注入 polishNote
     String extraNote = buildPolishContext(chapter, styleFingerprint)
     String originalNote = chapter.getPolishNote()
     chapter.setPolishNote(truncate(extraNote + "\n" + originalNote, 500))
     save(chapter)
     ctx.generateAndSave(POLISHING, chNum)
     // 恢复原始 polishNote
     chapter.setPolishNote(originalNote)
     save(chapter)
```

### 修改文件
`src/main/java/com/storycreator/workflow/engine/WorkflowContextBuilder.java`

### 变更

在设置 `chapterSummary` 的位置：

```java
// 原代码
context.setChapterSummary(co.getSummary());

// 改为
String summary = co.getSummary();
if (co.getEventPlan() != null && !co.getEventPlan().isBlank()) {
```

## 关键设计决策

| 决策 | 原因 |
|------|------|
| 不新增 WorkflowStep 枚举值 | PREPARATION 在策略内部实现，不影响现有状态机 |
| 中间结果存 ChapterEntity | 与章节 1:1 关系，避免额外 JOIN，简化查询 |
| 子步骤用阻塞式 AI 调用 | 在虚拟线程中安全，复用 ProofreadingService 已验证的模式 |
| eventPlan 追加到 chapterSummary | 无需新模板变量，向后兼容 DEFAULT |
| 条件性优化 (Step 5) | 审查通过则跳过，节省约 15% AI 调用 |
| 周期性深度审查不回溯修改 | 避免复杂度爆炸，仅记录并影响后续生成 |
| writingCycleStatus 实现断点续跑 | 任意步骤中断后可精确恢复，不重复已完成工作 |

## AI 调用成本对比

| 模式 | 每章调用次数 | 100 章总调用 | 预估时间/章 |
|------|-------------|-------------|------------|
| DEFAULT | ~3 次 | ~900 | ~2 分钟 |
| ENHANCED | ~7 次 | ~1100 | ~8-10 分钟 |

**成本增加约 20%，时间增加约 4-5 倍，但质量显著提升。**
