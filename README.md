# Story Creator - AI 网文创作工坊

AI 驱动的中文网络小说创作系统，提供从世界观设定到校对精修的全流程创作工作流。支持 Claude、OpenAI、Ollama 等多种大模型/协议，可一键全自动创作或逐步人工审核。

## 环境依赖

| 依赖 | 版本要求 | 说明 |
|------|---------|------|
| Java | 21+ | 运行时环境 |
| Maven | 3.8+ | 构建工具 |
| AI API Key | 至少一个 | Claude / OpenAI / Ollama（本地免 Key） |

数据库使用内嵌 H2，无需额外安装。

## 快速启动

```bash
# 方式一：后台运行（端口 1888，自动处理端口冲突）
./start_web.sh

# 方式二：前台运行（端口 8080）
mvn spring-boot:run

# 方式三：指定端口
mvn spring-boot:run -Dspring-boot.run.arguments="--server.port=1888"

# 打包为独立 jar
mvn package -DskipTests
java -jar target/story-creator-1.0.0-SNAPSHOT.jar
```

启动后访问 `http://localhost:8080`（或对应端口）即可使用。

## 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CLAUDE_API_KEY` | Anthropic API Key（备用，设置页优先） | 无 |
| `OPENAI_API_KEY` | OpenAI 或兼容代理 API Key（备用） | 无 |
| `STORY_DB_PATH` | H2 数据库存储目录 | `./data` |

## 模型配置

启动后进入 **设置页面**（`/settings`）配置 AI 模型：

1. 点击「添加模型配置」
2. 选择提供商（Claude / `OpenAI` / Ollama）
3. 填写模型 ID（如 `Qwen3.6-35B-A3B-Uncensored-Heretic-MLX-8bit`）
4. 填写 API Key 和 Base URL（支持自定义代理地址）, 如 oMLX的地址: `http://127.0.0.1:8000`
5. 可选填 Extra Params（JSON 格式，如 `{"enable_thinking": false}`）
6. 点击「测试连接」验证配置
7. 刷新页面，选择「全局默认模型」，并保存, 可指定默认模型

### 模型选择优先级（3 级级联）

```
步骤级覆盖 → 项目默认模型 → 全局默认模型 → 第一个激活的配置
```

每个工作流步骤可在界面中单独选择模型，实现不同步骤用不同模型。

### 支持的提供商

| 提供商 | Base URL 示例 | 说明 |
|--------|--------------|------|
| oMLX | `http://localhost:8000` | 本地或局域网部署, 选择 OpenAI 类型， API Key是自己掌握的 |
| Claude | `https://api.anthropic.com` | Anthropic 官方 API |
| OpenAI | `https://api.openai.com` | 官方或兼容代理（如 Azure、DeepSeek、通义千问等） |
| Ollama | `http://localhost:11434` | 本地部署，无需 API Key |

## 创建项目与创作流程

### 创建项目

1. 首页点击「新建项目」
2. 填写基本信息：
   - **标题**：小说名称
   - **类型**：玄幻 / 仙侠 / 都市 / 历史 / 言情 / 科幻 / 悬疑 / 奇幻 / 武侠 / 其他
   - **简介**：故事梗概（作为 AI 创作的种子）
   - **总章节数**：5–1000（默认 10）
   - **每卷章节数**：5–50（默认 10）
   - **角色数量**：1–20（默认 5）
   - **章节字数**：目标 / 最小 / 最大（默认 5000 / 4000 / 6000 字）
3. 可选：指定项目默认模型、启用全自动模式、从已有项目复制设置
4. 可选：为每个步骤填写「创作指导」（风格偏好、禁忌内容等）

### 创作工作流（6 步流水线）

进入项目的「工作流」页面，按顺序完成以下步骤：

| 步骤 | 名称 | 操作 |
|------|------|------|
| 1 | 世界观设定 | AI 生成世界背景 → 审阅编辑 → 确认 |
| 2 | 角色设计 | AI 生成角色总览 + 独立角色卡 → 可逐卡精修 → 确认 |
| 3 | 大纲生成 | AI 生成故事梗概 + 分卷纲要 + 逐章大纲 → 确认 |
| 4 | 章节写作 | 逐章生成正文（带入大纲、角色、前文上下文）→ 确认 |
| 5 | 润色修改 | 逐章润色（可附加修改意见）→ 确认 |
| 6 | 校对精修 | 生成校对报告（情节/角色/连续性/伏笔检查）→ 应用修正 |

每一步：**AI 生成 → 人工审阅/编辑 → 确认推进**。

### 全自动创作

创建项目时开启「Auto Mode」，点击「开始全自动创作」即可一键完成全部流程：
- 支持逐步骤启用/禁用
- 可随时停止，下次自动从断点续跑
- 实时查看生成内容流

## 功能列表

### 核心创作

- **6 步创作流水线**：世界观 → 角色 → 大纲 → 写作 → 润色 → 校对
- **全自动创作（AutoRun）**：一键执行全流程，断点续跑，协作式取消
- **后台生成**：SSE 流式输出，断线重连自动续读
- **角色卡系统**：独立角色卡生成 + AI 精修
- **角色状态追踪**：每章自动生成角色状态快照（修为、装备、位置、关系等 10 个维度）
- **上下文摘要**：AI 压缩世界观/角色信息，优化 prompt 长度
- **章节标题生成**：写作完成后自动为所有章节生成标题

### 模型与提示词

- **多模型支持**：Claude、OpenAI（及兼容 API）、Ollama
- **3 级模型级联**：步骤级 → 项目级 → 全局默认
- **提示词模板管理**：15 个子步骤模板，支持自定义编辑
- **类型感知模板**：按小说类型自动选择对应提示词
- **模板导入/导出**：JSON 格式批量管理

### 导出与导入

- **4 种导出格式**：Markdown、纯文本、PDF（中文字体支持）、EPUB
- **完整备份导出**：JSON 格式，包含所有项目数据
- **项目导入**：支持从 JSON 备份恢复，可选覆盖模式

### 其他功能

- **阅读器视图**：清爽的小说阅读界面
- **AI 用量统计**：按项目+模型追踪调用时长
- **全局替换**：跨章节批量替换文本
- **创作指导**：每步骤可配置风格方向和注意事项
- **10 种小说类型**：玄幻、仙侠、都市、历史、言情、科幻、悬疑、奇幻、武侠、其他
- **AI 超时配置**：全局可调（30–3600 秒，默认 300 秒）

### 校对检查项

- 情节摘要生成
- 角色姓名一致性检查
- 内容逻辑一致性检查
- 前后文连续性检查
- 伏笔回收检查
- 自动修正应用

## 技术栈

- **后端**：Java 21、Spring Boot 3.3、Spring WebFlux（响应式流）、JPA/Hibernate
- **数据库**：H2（文件模式）、Flyway 迁移管理
- **前端**：Thymeleaf + Alpine.js + Bootstrap 5.3
- **AI 通信**：WebClient 动态构建，按请求携带连接信息
- **并发**：Java 21 虚拟线程
- **导出**：OpenPDF（PDF）、epub4j（EPUB）
